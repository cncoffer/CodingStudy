# 8-12

计算小于n的质数.

# 算法一

首先实现函数IsPrimes, 然后对小于n的数进行遍历, 统计质数的数量.
时间复杂度为o(n^2)

当n=499979时, 超出时间限制.
说明需要更快的算法.

# 算法二

创建一个大小为n的数组初始化为true, 
然后依次将2 - n/2的倍数依次置为false.
最后遍历一遍数组, 所有仍为true的数即为质数.

时间复杂度也是o(n^2), 但应该比算法一的计算量会少一些.

既然这样的话两个就互有优劣了, 都实现吧.

算法二确实会快一些, 通过了测试.
